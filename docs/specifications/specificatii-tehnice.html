Specificatii Tehnice

Diagrama arhitecturii sistemului

Aspecte generale

Proiectul reprezinta un sistem complet de comandare, receptionare a comenzilor, si administrare a unui meniu de restaurant pizzerie. Componente si tehnologiile folosite:
  1. Baza de date - MySQL
  2. Servicii REST - Java Spring
  3. Aplicatia clientului - React Native
  4. Aplicatia de gestionare a comenzilor pentru casier - React Native
  5. Aplicatia de administrare a sistemului - React si Vite
Sistemul nu va trata livrarea comenzilor, aceasta fiind oferita de catre un tert. In functie de API-ul pus la dispozitie de catre acesta exista posibilitate integrarii unei harti in timp real pentru urmarirea livrarii. Din punctul de vedere al livrarii, este pus la dispozitie un computer intreg ce poate fi folosit pentru serverul de baza de date, serviciile REST si aplicatia web de administrare a sistemului.

Baza si modelul de date

  • foloseste MySQL
    • simplitate
    • aderare la standardele SQL
  • pune accent pe pastrarea unui istoric pe cat posibil de complet prin stergerea logica a datelor

Serviciile REST

  • foloseste Java ca limbaj si Spring ca framework de baza
    • limbaj OO simplu
    • mecanisme de IoC si DI pentru gestionarea logicii aplicatiei
    • documentatie abundenta
    • securitate dovedita
    • configuratie simpla
    • faciliteaza dezvoltarea pe modelul MVC
  • foloseste Spring Boot pentru a oferi servicii REST
    • server HTTP integrat
  • foloseste Spring Data JPA pentru maparea tabelelor bazei de date la entitati
    • contine HikariCP pentru a imbunatati performanta conectarii cu baza de date
    • reduce codul boilerplate pentru operatiile CRUD
    • mecanism de reflexie in baza de date a modificarilor aduse entitatilor
  • foloseste Spring Security pentru autentificare si autorizare

Aplicatia clientului

  • foloseste JavaScript ca limbaj si React Native, respectiv Expo ca framework
    • limbaj ce permite atat programare functionala, cat si obiectuala
    • limbaj ce permite creare dinamica a obiectelor, ceea ce ajuta la reducerea codului boilerplate de declarare a tipurilor de date
    • filozofia framework-ului este foarte similara cu cea a aplicatiilor web scrise cu React
    • componente UI similare aplicatiilor web
    • limbaj de markup JSX, limbaj de stilizare CSS
    • cross-platform
    • ecosistem mare si dezvoltat de pachete si biblioteci
    • performanta destul de buna pentru scopul aplicatiei
  • foloseste Axios pentru a face cereri HTTP catre server
    • API simplu, similar cu Fetch API
    • reduce din numarul de functionalitati ce ar trebui implementate manual in cazul folosirii Fetch API
    • interceptori pentru JWT si altele
  • foloseste TanStack Query in contextul cererilor HTTP
    • functionalitati de caching si refetching cu timeout, ce reduc numarul de cereri catre server si imbunatatesc performanta
    • implementare a starii erorii (loading, error, etc.) ce reduce codul boilerplatte
  • foloseste NativeWind pentru stilizare
    • similar cu Tailwind CSS
    • scurteaza limbajul de stilizare prin clase cu denumiri mai scurte pentru feature-uri de baza
    • posibilitatea declararii propriilor stiluri, culori, etc.
    • breakpoints pentru aplicarea conditionala a stilurilor in functiie de dimensiunea ecranului
    • utilizarea inline ajuta la gasirea rapida a contextului si modificarea lui pentru a rezolva probleme de randare sau a schimba comportamentul componentei in UI

Aplicatia de gestionare a comenzilor de catre casier

  • aplicatie pentru tableta Android
  • scrisa in React Native si Expo pentru un codebase uniform
  • se aplica toate tehnologiile folosite anterior la aplicatia clientului
  • foloseste react-native-bluetooth-escpos-printer pentru tiparirea bonurilor comenzilor

Aplicatia de administrare a sistemului

  • aplicatie web scrisa in React si Vite
    • aceleasi dependente cu aplicatia clientului
  • foloseste TanStack React Table pentru afisarea datelor
    • eficienta ridicata
    • suport pentru paginare, filtrare, sortare
    • API bazat pe React Hooks

Performanta - limite superioare

throughput 100 de clienti simultan
timp de raspuns pentru o conexiune buna a clientului la internet 5s
dimensiunea bazei de date 10 GB
